Роботу виконав Мунчак Євген 471
1) Призначення

Бот “Організатор подій” допомагає додавати, переглядати, шукати та видаляти події студента (лекції, зустрічі, іспити, нагадування тощо). Працює через консоль, команди вводяться текстом.

2) Як зберігаються дані

Події зберігаються у JSON-файлі events.json у тій самій папці, де лежить програма.
Кожна подія має структуру:
id (унікальний номер)
title (назва)
date (дата у форматі YYYY-MM-DD)
time (час HH:MM або порожньо)
type (тип події: lecture/meeting/exam/reminder/custom)
note (довільна нотатка)
created_at (коли створено)

3) Як обробляються команди

Програма працює в циклі: читає рядок користувача input("> ").
Рядок розбивається на:
назву команди (перше слово),
аргументи (решта тексту).
Для кожної команди є окрема функція-обробник (наприклад handle_add, handle_list тощо).
Якщо команда невідома — бот показує підказку help.

4) Загальний алгоритм роботи (список кроків)

При старті програма намагається завантажити події з events.json.
Показує коротке вітання та підказку help.
Запускає нескінченний цикл:
зчитує команду користувача;
викликає відповідну функцію-обробник;
за потреби змінює список подій;
після змін зберігає дані назад у events.json.
Завершує роботу по команді exit.

5) Підтримувані команди

help — список команд

add — додати подію (бот задає питання)

list — показати всі події (відсортовано)

today — події на сьогодні

next — найближчі події (за замовчуванням 5; можна next 10)

search <текст> — пошук за назвою/нотаткою

delete <id> — видалити подію за ID

clear — видалити всі події (попросить підтвердження)

exit — вихід
